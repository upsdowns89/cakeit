
# 🎂 서비스 기획서: Caki (케키) - 커스텀 케이크 주문 중개 플랫폼

## 1. 서비스 개요

* **서비스명:** Caki (케키)
* **목적:** 카카오톡의 번거로운 텍스트 주문 방식을 탈퇴하고, 시각적인 도안 선택과 구조화된 양식을 통해 소비자와 판매자를 연결하는 커스텀 케이크 전용 플랫폼.
* **핵심 가치:** * 소비자: 디자인 선택의 편의성 및 주문 상태 확인의 투명성.
* 판매자: 주문서 관리 효율화 및 작업 큐(Queue) 시각화.



---

## 2. 메뉴 구조 (Information Architecture)

### [소비자 모드]

1. **홈 (둘러보기):** 지역별 검색, 지도 보기, 카테고리별 인기 가게.
2. **가게 상세:** 가게 소개, 포트폴리오(사진), 가격표, **주문하기 버튼**.
3. **주문하기 (Step Form):** 도안 선택(프리셋/업로드) -> 옵션 선택 -> 정보 입력.
4. **내 주문:** 주문 진행 상태 확인, 판매자와의 1:1 채팅 스레드.
5. **마이페이지:** 프로필 수정, 찜한 가게, 작성한 리뷰.

### [판매자 모드]

1. **대시보드 (주문 QUE):** 신규/진행중/완료 주문을 칸반 보드 형태로 관리.
2. **스케줄 관리:** 날짜별 예약 가능 여부 설정 (캘린더 뷰).
3. **가게 관리:** 가게 정보, 메뉴 및 가격, 주문서 양식 커스텀 설정.
4. **채팅 목록:** 고객별 주문 상담 스레드 관리.

---

## 3. 상세 화면 기획 (Wireframe Specification)

### 3.1. 가게 둘러보기 및 검색

* **기능:** 현재 위치 기반 주변 가게 노출, '강남구', '홍대' 등 지역 필터.
* **UI 요소:** 가게 썸네일 카드, 별점, 최소 가격 정보, 지도 토글 버튼.

### 3.2. 스마트 주문서 작성 (핵심 기능)

* **Step 1. 디자인 선택:** * 프리셋 이미지(생일, 퇴사, 짤 등) 격자형 노출.
* 직접 그린 도안 이미지 업로드 영역.


* **Step 2. 케이크 상세 옵션:**
* 사이즈(도시락, 1호, 2호), 시트 맛, 크림 종류 선택 (Radio/Select box).


* **Step 3. 텍스트 및 일정:** * 레터링 문구 입력 칸.
* 픽업 날짜 및 시간 선택 (Date Picker).



### 3.3. 주문 상태 및 스레드 (채팅)

* **상태 정의:** `대기` -> `수락` -> `입금대기` -> `제작중` -> `픽업대기` -> `완료`.
* **채팅창:** 주문서 내용이 상단에 고정된 상태로 하단에서 판매자와 실시간 대화.

## 4. 백엔드 설계 (Supabase 기반)

Supabase의 **PostgreSQL, Auth, Storage**를 활용한 구조입니다.

### 4.1. 데이터베이스 테이블 (Schema)

| 테이블명 | 설명 | 주요 컬럼 |
| --- | --- | --- |
| `profiles` | 사용자 공통 정보 | id(PK), email, nickname, role(buyer/seller), phone |
| `shops` | 판매자 가게 정보 | id(PK), seller_id(FK), name, address, description, lat, lng |
| `cake_presets` | 도안 프리셋 | id(PK), category, image_url, title |
| `orders` | 주문서 정보 | id(PK), buyer_id, shop_id, status, pickup_date, total_price, design_img_url |
| `order_options` | 주문 세부 선택사항 | id(PK), order_id(FK), option_name, option_value |
| `messages` | 채팅 메시지 | id(PK), order_id(FK), sender_id(FK), content, created_at |
| `reviews` | 고객 리뷰 | id(PK), order_id, buyer_id, rating, comment, photo_url |

### 4.2. Storage 구성 (이미지 저장)

* `shop-images/`: 가게 프로필 및 포트폴리오 사진.
* `order-designs/`: 고객이 업로드한 커스텀 도안.
* `chat-files/`: 채팅 중 주고받는 이미지.

## 5. 추가 추천 기능 (Vibe Coding 보완점)

1. **실시간 예약 캘린더 (판매자용):**
* 특정 날짜에 주문이 꽉 차면 자동으로 '주문 불가' 처리가 되도록 하여 판매자의 피로도를 낮춤.


2. **자동 입금 확인 연동 알림:**
* 입금 계좌 정보를 채팅창 상단에 띄우고, 입금 확인 버튼 클릭 시 구매자에게 푸시 알림 발송.


3. **나만의 도안 캔버스 (선택 사항):**
* 간단한 그리기 도구(Canvas API)를 웹 내에 제공하여 외부 앱 없이 도안을 그려서 바로 전송.
