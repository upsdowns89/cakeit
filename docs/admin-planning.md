# 프로젝트: 커스텀 케이크 플랫폼 [사장님용 관리 어드민] 구축

## 1. 배경 및 목적
- 카카오톡 수동 주문의 불편함을 해소하고, 사장님이 웹/모바일에서 직접 주문 상세 내용을 수정하고 스케줄을 관리할 수 있는 시스템 구축.
- DB: Supabase (Table: shops, orders, products, slots 사용)

## 2. 핵심 기능: 주문서 수정 및 유연한 가격 대응 (Critical)
- **주문 수정 로직:** 고객이 처음 보낸 주문서 내용을 사장님이 상담 후 수정할 수 있어야 함.
  - `orders` 테이블의 각 컬럼(맛, 사이즈, 픽업시간 등)을 사장님 화면에서 `Editable`하게 구현.
  - 수정 시 총 금액(`total_price`)이 자동 재계산되거나 사장님이 직접 최종가를 입력할 수 있는 필드 제공.
- **입금 상태 동기화:** 사장님이 [입금 확인] 클릭 시 `status`를 'Confirmed'로 변경하고, Supabase Real-time으로 고객 앱에 즉시 반영.

## 3. 화면별 상세 기획

### [화면 1: 대시보드]
- 오늘 픽업 예정인 케이크 리스트를 시간순(Timeline)으로 카드 배치.
- 카드 내 필수 정보: 고객명, 케이크 타입, 픽업 시간, 입금 상태.

### [화면 2: 주문서 상세 및 편집기]
- 좌측: 고객 제출 디자인 이미지 및 요청 사항.
- 우측: 편집 가능한 주문서 폼 (Select Box, Text Input 구성).
- 하단: [금액 수정] 및 [주문 업데이트] 버튼. 
- *알림 기능:* 수정 완료 시 고객에게 변경 사항을 안내하는 시스템 메시지 트리거 포함.

### [화면 3: 스케줄 및 슬롯 관리]
- 캘린더 UI 내에 날짜별 예약 건수 표시.
- 특정 날짜 클릭 시 해당 날짜의 '예약 가능 수량(Capacity)'을 즉시 변경하거나 마감(Close)할 수 있는 토글 UI.

### [화면 4: 채팅 & 주문 연동]
- 1:1 채팅창 우측 사이드바에 현재 채팅 중인 고객의 '최신 주문서'를 요약해서 노출.
- 상담 도중 바로 주문서 상세 페이지로 이동할 수 있는 바로가기 버튼.

## 4. 기술적 요구사항 (Developer Note)
- **Responsive Design:** 주방 작업 중 스마트폰으로 확인하는 경우가 많으므로 모바일 최적화 필수.
- **State Management:** 주문 정보 수정 시 즉시 상태를 업데이트하고 DB와 동기화.
- **Security:** Supabase Auth 및 RLS를 적용하여 로그인을 완료한 사장님만 본인 가게의 데이터에 접근 가능하도록 설계.

위 기획안을 바탕으로 직관적이고 현대적인 관리자 UI와 비즈니스 로직을 생성해줘.
